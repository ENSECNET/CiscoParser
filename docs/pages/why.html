Why a parser

Documenting a network by hand drifts the day it is written. The device's config does not.

The problem

A running network device knows everything about itself. Its running-config is the single most authoritative description of how it is set up: interfaces, addresses, VLANs, VRFs, routing, authentication, logging.

But that knowledge lives only on the device — flat text, in a vendor dialect, on hundreds of boxes. The moment you want a picture — an inventory, an IPAM, an audit — you are retyping what the devices already know. The spreadsheet is stale before you save it; the config keeps changing.

Why it is useful

  • Inventory without typing. Point it at a config and the device, its interfaces, addresses and VLANs appear in NetBox.
  • IPAM that matches reality. Addresses come from the config, not from memory.
  • Documentation you can re-run. When a config changes, re-parse it — the model is regenerated from ground truth and cannot silently drift.
  • Audit and review. A normalized view makes it easy to spot the interface with no description, the VLAN defined but unused, the ACL that does not belong.
  • A foundation for everything downstream. Once the network lives in NetBox as structured data, diagrams, compliance checks and automation all have something to read from.

Where it fits

CiscoParser is an import / documentation tool, not a configuration manager. It takes the device's own truth and turns it into a model you can keep. It pairs naturally with a standalone NetBox deployment: NetBox is the store, CiscoParser is one of the doors data comes in through.

Načo parser

Ručná dokumentácia siete sa rozchádza s realitou v deň, keď vznikne. Config zariadenia nie.

Problém

Bežiace sieťové zariadenie o sebe vie všetko. Jeho running-config je najautoritatívnejší popis toho, ako je nastavené: rozhrania, adresy, VLAN-y, VRF-y, smerovanie, autentifikácia, logovanie.

Lenže táto znalosť žije len na zariadení — ako plochý text, vo vendorskom dialekte, na stovkách boxov. Keď chceš obraz — inventár, IPAM, audit — prepisuješ to, čo zariadenia už vedia. Tabuľka je neaktuálna skôr, než ju uložíš; config sa stále mení.

Prečo je užitočný

  • Inventár bez písania. Načítaš config a zariadenie, jeho rozhrania, adresy a VLAN-y sa objavia v NetBoxe.
  • IPAM, ktorý sedí s realitou. Adresy pochádzajú z configu, nie z pamäte.
  • Dokumentácia, ktorú vieš spustiť znova. Keď sa config zmení, naparsuješ ho znova — model sa regeneruje zo zdroja pravdy a nemôže ticho zostarnúť.
  • Audit a revízia. Normalizovaný pohľad uľahčí nájsť rozhranie bez popisu, VLAN definovanú no nepoužitú, ACL, ktorá tam nepatrí.
  • Základ pre všetko ďalšie. Keď sieť žije v NetBoxe ako štruktúrované dáta, diagramy, compliance kontroly aj automatizácia majú z čoho čítať.

Kam zapadá

CiscoParser je nástroj na import / dokumentáciu, nie správca konfigurácie. Berie vlastnú pravdu zariadenia a mení ju na model, ktorý si uchováš. Prirodzene sa páruje so samostatným NetBox nasadením: NetBox je úložisko, CiscoParser sú jedny z dverí, ktorými dáta vstupujú.
