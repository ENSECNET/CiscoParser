Deploy on Proxmox

One thin deployer: minimal Debian 12 LXC → Docker → clone repo → compose up.

curl -fsSL https://raw.githubusercontent.com/ensecnet/CiscoParser/main/deploy/lxc-deploy.sh -o lxc-deploy.sh
bash lxc-deploy.sh

The wizard asks for container ID, hostname, storage, network (bridge / VLAN / IP) and the web port, then builds everything. Open http://<lxc-ip>:8080.

Any Docker host

cp .env.example .env        # optionally set NETBOX_URL / NETBOX_TOKEN
docker compose up -d --build
# UI: http://localhost:8080

Update

pct exec <CTID> -- bash -c 'cd /opt/ciscoparser && git pull && docker compose up -d --build'
NetBox tokens (4.5+) are shown once at creation. Copy the full key immediately — it cannot be retrieved later.

Nasadenie na Proxmox

Jeden tenký deployer: minimal Debian 12 LXC → Docker → clone repo → compose up.

curl -fsSL https://raw.githubusercontent.com/ensecnet/CiscoParser/main/deploy/lxc-deploy.sh -o lxc-deploy.sh
bash lxc-deploy.sh

Sprievodca sa spýta na container ID, hostname, úložisko, sieť (bridge / VLAN / IP) a web port, potom všetko postaví. Otvor http://<lxc-ip>:8080.

Ľubovoľný Docker host

cp .env.example .env        # voliteľne nastav NETBOX_URL / NETBOX_TOKEN
docker compose up -d --build
# UI: http://localhost:8080

Aktualizácia

pct exec <CTID> -- bash -c 'cd /opt/ciscoparser && git pull && docker compose up -d --build'
NetBox tokeny (4.5+) sa pri vytvorení zobrazia raz. Skopíruj plný kľúč hneď — neskôr sa nedá získať.
