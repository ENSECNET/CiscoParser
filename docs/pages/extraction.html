CiscoParserENSECNET · network docs

What it extracts

Native objects become real NetBox records; everything else is preserved verbatim as config context.

CiscoParser reads the config the way an engineer does — it understands the hierarchical parent/child structure (interface → ip address → shutdown), not just lines of text.

Native NetBox objects

From configBecomes
hostnameDevice name
interface …Interface (type, description, enabled state)
ip address A BIP address (mask → CIDR)
vlan NVLAN
ip vrf / vrf definitionVRF (+ route distinguisher)

Config context (textual)

Everything NetBox does not model natively is captured verbatim and attached to the device as local config context, so nothing is lost:

GroupMatches
routingOSPF, EIGRP, BGP, static routes
dhcpDHCP pools
acls / nataccess-lists, NAT rules
snmp / ntp / loggingmanagement services
aaaAAA, TACACS+, RADIUS, local users
crypto / qos / linesPKI/IPsec, QoS policy, console/VTY
The split is deliberate: things NetBox models natively become records you can query, filter and relate; everything else is preserved so the device's full configuration stays attached.

Čo extrahuje

Natívne objekty sa stanú reálnymi NetBox záznamami; všetko ostatné sa zachová doslovne ako config context.

CiscoParser číta config tak ako inžinier — rozumie hierarchickej štruktúre rodič/dieťa (interface → ip address → shutdown), nielen riadkom textu.

SkupinaZachytáva

Rozdelenie je zámerné: čo NetBox modeluje natívne, stane sa záznamom, ktorý vieš filtrovať a prepájať; všetko ostatné sa zachová, aby zostala pripojená plná konfigurácia zariadenia.
